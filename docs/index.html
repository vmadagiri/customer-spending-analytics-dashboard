Project Overview

This project simulates a real business analytics workflow for a retail company. It starts with synthetic operational data, loads the data into a normalized PostgreSQL database, answers stakeholder questions with SQL, exports reusable analysis outputs with Python, and presents the final insights in an interactive Streamlit dashboard.

The portfolio deliverable includes both a working analytics application and this Quarto case study, so recruiters and interviewers can quickly understand the business problem, technical approach, and decision-making value.

Business Problem

Retail leaders need to know where growth is coming from, which customers are worth retaining, how product categories perform, and whether discounts are improving revenue without damaging profitability. The goal of this analysis is to convert transaction-level data into recommendations for merchandising, retention, regional strategy, and dashboard-driven decision-making.
